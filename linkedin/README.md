# LinkedIn OS

Canonical repository for the entire LinkedIn presence. The live LinkedIn profile is a **deployment target** of these files — edits happen here first, then get applied to LinkedIn, never the other way around.

Framing derives from [`docs/career-positioning.md`](../docs/career-positioning.md) and [`docs/professional-brand.md`](../docs/professional-brand.md). Facts derive from [`docs/career-history.md`](../docs/career-history.md) and the experience records. The rules that bind every file here:

- No unverified metrics (Tier C in [`docs/achievement-library.md`](../docs/achievement-library.md)) anywhere on LinkedIn.
- Year-only dates for fuzzy older roles (Alorica, UnitedHealthcare), per the Resolved Decisions in `career-history.md`.
- Voice rules from `professional-brand.md`: plain over impressive, evidence-humble, no hype vocabulary.
- Client-identifying content follows the gates in [`docs/case-studies.md`](../docs/case-studies.md).

## Files

### Profile surfaces (what the profile says)

- [`headline.md`](headline.md) — headline options, recommendation, and keyword rationale
- [`about.md`](about.md) — the About section, deploy-ready
- [`experience.md`](experience.md) — every Experience entry, deploy-ready
- [`featured.md`](featured.md) — Featured section plan and gating
- [`recommendations.md`](recommendations.md) — who to ask, what they can attest to, request templates

### Growth surfaces (how the profile compounds)

- [`creator-positioning.md`](creator-positioning.md) — what to be known for and to whom
- [`content-pillars.md`](content-pillars.md) — the recurring themes posts draw from
- [`post-frameworks.md`](post-frameworks.md) — reusable post structures with guardrails
- [`connection-messages.md`](connection-messages.md) — connection-request templates by audience
- [`outreach.md`](outreach.md) — outbound sequences for consulting and job-search pipelines
