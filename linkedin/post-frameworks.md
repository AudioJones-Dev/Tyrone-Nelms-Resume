# Post Frameworks

Reusable structures for LinkedIn posts. Every post uses one of these skeletons, draws from a pillar in [`content-pillars.md`](content-pillars.md), and passes the guardrails at the bottom.

## Framework 1 — Story → Lesson (STAR-lite)

Best for Pillar 3 (operator lessons). Source stories from [`docs/interview-playbook.md`](../docs/interview-playbook.md) — they are already evidence-checked.

```
[One-line scene from a real role — specific, sensory, verified]
[What went wrong or what the pressure was]
[What I did — 2-3 short lines]
[The transferable principle for service businesses today]
[One-line question to readers]
```

## Framework 2 — Before / After Workflow

Best for Pillars 1–2. Qualitative until case-study gates clear.

```
[The "before" state — messy, recognizable, anonymized]
[The one change that mattered (not the tool — the design decision)]
[The "after" state — described qualitatively]
[Why the obvious alternative fails]
```

## Framework 3 — Myth vs. Practice

Best for Pillar 1 (AI hype correction).

```
["Everyone says X about AI/automation"]
[What actually happens in a service business when you try X]
[What works instead — grounded in operational reality]
```

## Framework 4 — The Checklist

Best for Pillar 2. High-save-rate format.

```
[Problem statement in one line]
[5-8 checklist items, each one actionable line]
[Which item most people skip, and the cost]
```

## Framework 5 — Build-in-Public Update

Pillar 4 only (gated — see `content-pillars.md`).

```
[What changed in the Career OS this week]
[The governance decision behind it (gates, tiers, verification)]
[The general principle any professional could apply]
```

## Guardrails (every post, no exceptions)

1. **No Tier C claims.** Check [`docs/achievement-library.md`](../docs/achievement-library.md) — if the number isn't Tier A, the post goes qualitative.
2. **Client gates.** Any client-adjacent story must pass the anonymization rules in [`docs/case-studies.md`](../docs/case-studies.md) — and anonymized still means evidence-backed.
3. **Voice check.** Plain over impressive; banned vocabulary list in `docs/professional-brand.md`.
4. **Consistency check.** Would this post make sense coming from the About section? If not, it doesn't ship.
5. **No engagement bait.** No "agree?", no fake vulnerability, no manufactured controversy.

## Comment Strategy

Commenting is content. Default behavior: substantive comments on posts by peer operators and prospective-audience founders — add an operational detail or a respectful counter-example, never "Great post!"
